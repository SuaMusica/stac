# STAC CLI

A command-line tool for building STAC (Static Type-Aware Components) DSL files into JSON format and deploying them to STAC cloud for Flutter applications.

## Overview

The STAC CLI tool analyzes your STAC DSL files written in Dart and converts them into JSON format that can be consumed by the STAC framework at runtime. It also provides deployment capabilities to upload your STAC components to the cloud for dynamic loading.

## Features

- 🔍 **Automatic Discovery**: Finds all `.dart` files in your `stac/` directory
- 🧠 **Smart Analysis**: Uses Dart analyzer to identify functions returning `StacWidget`
- ⚡ **Fast Execution**: Processes functions in isolated environments
- 📁 **Organized Output**: Generates JSON files in `.stac_build/` directory
- 🌐 **Cloud Deployment**: Deploy JSON files to STAC cloud via Firebase
- 🎨 **Beautiful Output**: User-friendly CLI with emojis and progress indicators
- 🐛 **Verbose Mode**: Detailed logging for debugging
- ⚙️ **Configuration Support**: Store settings in `.stacrc` file

## Installation

### Global Installation

```bash
dart pub global activate --source path packages/stac_cli
```

After global installation, you can use the `stac` command from anywhere:

```bash
stac build
stac deploy
```

### Local Usage

You can also run the CLI directly without global installation:

```bash
dart run packages/stac_cli/bin/stac_cli.dart build
dart run packages/stac_cli/bin/stac_cli.dart deploy
```

## Usage

### Build Command

```bash
stac build
```

This command will:
1. Look for a `stac/` directory in your project root
2. Find all `.dart` files in the directory
3. Analyze each file for functions returning `StacWidget`
4. Execute those functions and call their `toJson()` method
5. Save the resulting JSON to `.stac_build/` directory

### Deploy Command

```bash
stac deploy
```

Deploy your built JSON files to STAC cloud:

#### Command Line Options

```bash
# Deploy with specific parameters
stac deploy --project-id my-project --user-id cli-user

# Deploy with custom endpoint
stac deploy --project-id my-project --url https://custom-api.com/upsertJson

# Verbose deployment
stac deploy --verbose
```

#### Available Options

- `--project-id` or `-p`: Project ID
- `--user-id` or `-k`: User ID for deployment
- `--url` or `-u`: Custom API endpoint URL

### Configuration File

Create a `.stacrc` file in your project root to store deployment settings:

```json
{
  "project_id": "my-project",
  "user_id": "cli-user",
  "api_url": "https://us-central1-stac-7ad77.cloudfunctions.net/upsertJson"
}
```

With a config file, you can simply run:

```bash
stac deploy
```

**Note**: Command line options override config file settings.

### Verbose Mode

For detailed output and debugging information:

```bash
stac build --verbose
stac deploy --verbose
```

### Help

```bash
stac --help
stac build --help
stac deploy --help
```

## Directory Structure

Your project should have the following structure:

```
your_flutter_project/
├── pubspec.yaml
├── .stacrc                   # Optional config file
├── stac/
│   ├── home_screen.dart
│   ├── profile_screen.dart
│   └── .stac_build/          # Generated by CLI
│       ├── home_screen.json
│       └── profile_screen.json
└── lib/
    └── ...
```

## STAC DSL Example

Here's an example of a STAC DSL file (`stac/home_screen.dart`):

```dart
import 'package:stac_models/core/stac_model.dart';
import 'package:stac_models/painting/stac_color/stac_colors.dart';
import 'package:stac_models/types/stac_double.dart';
import 'package:stac_models/widgets/widgets.dart';

StacWidget homeScreen() {
  return StacContainer(
    color: StacColors.pink,
    width: StacDouble(200.0),
    height: StacDouble(200.0),
    child: StacText(
      data: 'Hello World',
    ),
  );
}
```

After running `stac build`, this generates `stac/.stac_build/home_screen.json`:

```json
{
  "color": "#FFE91E63",
  "width": 200.0,
  "height": 200.0,
  "child": {
    "data": "Hello World",
    "type": "text"
  },
  "type": "container"
}
```

## Backend Setup (Firebase Cloud Functions)

The CLI works with the STAC cloud backend using the `upsertJson` endpoint. Each STAC JSON file is deployed individually as a "screen" with the following structure:

### API Request Format

```bash
curl -X POST https://us-central1-stac-7ad77.cloudfunctions.net/upsertJson \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "my-project",
    "screenName": "home_screen", 
    "jsonString": "{\"type\":\"container\",\"color\":\"#FFE91E63\"}",
    "userId": "cli-user"
  }'
```

### How it Works

1. **File Processing**: CLI converts `home_screen.json` → "home_screen" 
2. **JSON String**: File contents are sent as a string in `jsonString` field
3. **Screen Creation**: Each file becomes a separate screen in your project
4. **User Attribution**: Deployments are tracked by `userId`

## Requirements

- Functions must be top-level (not class methods)
- Functions must return a type that extends `StacWidget`
- Functions should be pure (no side effects) for reliable conversion
- All dependencies must be properly imported

## Error Handling

The CLI provides clear error messages for common issues:

- **Project not found**: Make sure you're in a Flutter project directory with `pubspec.yaml`
- **STAC directory not found**: Create a `stac/` directory in your project root
- **No StacWidget functions**: Ensure your functions return types extending `StacWidget`
- **Execution errors**: Check that your functions are pure and all dependencies are available
- **Build directory not found**: Run `stac build` before deploying
- **Authentication errors**: Check your user ID and project configuration
- **Project not found**: Ensure the project exists in your STAC cloud backend

## Integration with Flutter Apps

After deploying, you can load STAC components in your Flutter app:

```dart
import 'package:stac/stac.dart';

// Load from local build
Widget buildFromLocal(BuildContext context) {
  return Stac.fromAssets(
    'stac/.stac_build/home_screen.json',
    context: context,
  );
}

// Load from cloud (after deployment)
Widget buildFromCloud(BuildContext context) {
  return Stac.fromNetwork(
    context: context,
    request: StacNetworkRequest(
      url: 'https://your-api.com/stac/home_screen',
      method: 'GET',
    ),
  );
}
```
